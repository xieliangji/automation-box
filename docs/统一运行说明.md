# 统一运行说明

## 唯一路径

仓库已收敛为唯一路径，后续开发和问题修复统一基于：

- 入口：`main.py`
- 主类：`smart_monkey/app_runtime.py::SmartMonkeyAppRuntime`
- 输出：`output/run/`

历史分支入口已清理，后续统一通过 `main.py` 执行，不再维护并行入口脚本。

## 运行方式

1. 复制配置

```bash
cp config.example.yaml config.yaml
```

Windows PowerShell：

```powershell
copy config.example.yaml config.yaml
```

2. 修改目标应用

- `app.platform`
- `app.target_app_id`
- `app.launch_target`
- `app.package_name`
- `app.launch_activity`

其中 `target_app_id/launch_target` 是平台中性字段；`package_name/launch_activity` 为兼容字段。

3. 运行

```bash
python main.py
```

Windows PowerShell：

```powershell
python main.py
```

查看报告（Windows PowerShell）：

```powershell
start .\output\run\report\index.html
```

## 当前能力

- 状态建模与层级树解析
- 候选动作提取与高风险过滤
- 动作评分与探索策略
- watchdog 监控与 issue 落盘
- periodic screenshot
- replay 导出与 issue replay
- checkpoint 管理与 backtrack
- navigation recovery + recovery validation
- recorder index
- html / markdown 报告
- recovery metrics
- 服务化编排（recovery / report / telemetry / watchdog / orchestration）
- 会话护栏（session guardrails）：在登录/表单等高风险会话状态下抑制 `back/restart_app`，降低回登录循环概率
- 登录引导（login bootstrap）：可选自动填充账号密码并触发登录，减少停留登录页的预算浪费
- 目标导向探索开关（goal-directed switch）：`prefer_functional_pages` 默认开启，优先功能页；关闭后可做登录/账号链路小范围 Monkey
- 恢复策略升级（recovery upgrade）：当 `prefer_functional_pages=true` 时，恢复优先选择非登录 checkpoint，并过滤会导致回登录链路的回放动作
- 覆盖率基准评估（coverage benchmarking）：支持读取 baseline run 并输出 KPI 对比（`coverage_benchmark.json`）
- 运行 profile（`run.profile`）：支持 `functional` / `crash_stress` / `monkey_compatible`（默认 `functional`）
- Monkey 兼容增强模式（`run.profile=monkey_compatible`）：在当前主线上增加“防脱 + 防卡 + 风险冷却”的随机扰动增强（Android 先行）
- 双指缩放手势（pinch）：支持 `pinch_in` / `pinch_out` 动作（可通过 `policy.enable_pinch` 开关）

## Profile 说明

- `functional`：默认模式，行为与现有主线一致，强调稳定探索与恢复质量。
- `crash_stress`：高压模式，降低动作后等待并启用 burst 提权，提升 crash 暴露概率。
- `monkey_compatible`：Monkey 风格增强模式，重点强化防脱、防卡循环、风险动作冷却与轻量随机抖动。

策略组合关系（面向利益相关方）：

- 同一轮运行只能选择一个 `run.profile`。
- `crash_stress` 与 `monkey_compatible` 是互斥关系，不会并行生效。
- `learning.enabled` 是独立叠加层，可与任一 profile 组合使用。
- `monkey_compatible` 保持主线动作框架不变，仅在评分层追加 Monkey 风格调分。

业务影响解读：

- 选择 `functional`：优先稳定覆盖与恢复质量。
- 选择 `crash_stress`：优先压测强度与 crash 曝光效率。
- 选择 `monkey_compatible`：优先防脱/防卡与随机探索平衡。

- Android + `monkey_compatible`：`unique_states` 相对 baseline >= 1.25；`out_of_app_ratio` 退化 <= 0.02；`recovery_success_rate >= 0.95`。
- Android + `monkey_compatible` 且 `sidecar.monkey.enabled=true`：额外要求 `sidecar_batch_count >= 1`、`sidecar_success_rate >= 0.95`、`sidecar_recovery_failure_rate <= 0.2`。
- iOS（`functional`/`monkey_compatible`）：`unique_states` 相对 baseline >= 2.0；`out_of_app_ratio <= 0.03`；`recovery_success_rate >= 0.95`；`permission_like_step_ratio` 不高于 baseline。

gate 结果统一输出到 `report/coverage_benchmark.json -> gates`。

## Learning（Bandit）说明

- `learning.enabled=false` 时，保持原有规则打分行为。
- 启用后采用混合分：
  - `final_score = rule_score * alpha + learned_ucb * (1 - alpha)`
- 当前 learned 分基于 UCB（每个 arm 按历史奖励与探索置信项计算）。

## 代码组织约定

- 运行编排职责优先收敛在 `smart_monkey/services/`。
- `app_runtime.py` 作为运行壳层，避免在此堆叠细碎实现。
- 新能力优先接入主线链路，不再新增并行 `main_*` 入口。
